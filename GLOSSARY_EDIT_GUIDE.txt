코딩용어 모음집 수정 가이드

1) 용어 데이터 위치
- public/glossary-data.json

2) 새 용어 추가 방법
- glossary-data.json 맨 마지막 ] 바로 앞에 아래 형태로 한 항목 추가하세요.

{
  "slug": "new-term",
  "termKo": "새 용어",
  "termEn": "New Term",
  "category": "CATEGORY",
  "shortDescription": "짧은 한 줄 설명",
  "description": "조금 더 자세한 설명",
  "example": "const example = true;",
  "related": ["variable", "function"]
}

3) 꼭 지켜야 할 규칙
- slug: 영어 소문자 + 하이픈 형태로 작성
  예) object-oriented, async-await
- related: 이미 존재하는 slug를 넣어야 정상 연결됩니다.
- example: 줄바꿈이 필요하면 \n 으로 넣어도 되고, 실제 줄바꿈으로 넣어도 됩니다.

4) 상세 페이지 파일
- views/glossary-detail.html
- 이 파일 하나가 /glossary/:slug 형태의 모든 상세 페이지를 자동으로 보여줍니다.
- 즉, HTML 파일을 100개 따로 만들 필요가 없습니다.

5) 목록 페이지 파일
- views/glossary.html
- 검색창, 카테고리 버튼, 카드 목록 화면을 담당합니다.

6) 메뉴바 수정 위치
- views/index.html
- views/practice.html
- views/lecture.html
- views/history.html

7) 서버 라우트
- server.js
- /glossary
- /glossary/:slug

8) 로컬 실행
- npm install
- npm start
- 브라우저에서 http://localhost:5000 접속

9) Render 반영
- 수정 후 GitHub에 push
- Render가 연결된 저장소면 자동 재배포됨

10) 많이 하는 수정 예시
- 용어 이름 바꾸기: glossary-data.json에서 termKo, termEn 수정
- 설명 바꾸기: shortDescription, description 수정
- 예제 바꾸기: example 수정
- 카테고리 바꾸기: category 수정
